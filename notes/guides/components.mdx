This is the visual test page for Grimoire. Every built-in and custom component
appears below under its own heading with a short note and a live example — if
something renders wrong here, the build is broken.

## Callout

Boxed asides for emphasis. One per `type`, each with optional `title`.

<Callout type="note">
A plain **note** — neutral context that doesn't fit the main flow.
</Callout>

<Callout type="info" title="Heads up">
Informational detail. Markdown works inside: `inline code`, [links](https://example.com), and **bold**.
</Callout>

<Callout type="tip" title="Pro tip">
Prefer fenced code blocks with a language so syntax highlighting kicks in.
</Callout>

<Callout type="success" title="Shipped">
The 1.4.0 release passed all 312 checks and deployed in 41 seconds.
</Callout>

<Callout type="warning" title="Careful">
Editing `config.toml` while the dev server runs forces a full reload.
</Callout>

<Callout type="danger" title="Destructive">
`grimoire purge --all` deletes the rendered cache and cannot be undone.
</Callout>

## Steps

Ordered, numbered procedures. Each `Step` takes a `title`.

<Steps>
  <Step title="Scaffold">
Run `grimoire init my-grimoire` to create the folder layout and a sample note.
  </Step>
  <Step title="Write">
Add `.mdx` files under `notes/`. Frontmatter drives the title, tags, and order.
  </Step>
  <Step title="Preview">
`grimoire dev` serves the app with hot reload at `localhost:4321`.
  </Step>
  <Step title="Ship">
`grimoire build` emits a single static binary you can drop on any host.
  </Step>
</Steps>
