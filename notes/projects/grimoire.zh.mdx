你正在阅读的这篇笔记，正是由它所描述的系统渲染出来的。**Grimoire** 把一个装满 MDX 文件的
文件夹变成一个快速、可搜索的小应用——然后把整个应用连同运行时一起，折叠进一个独立的二进制文件，
让你可以把它拷贝到任何机器上。

## 核心主张

写 Markdown。当散文不够用时，丢进一个组件——一个 `<Chart/>`、一个
`<DataTable/>`、一个 `<Quiz/>`——无需导入，无需接线。构建过程会静态渲染每一篇
笔记，打包好可交互的「孤岛」，再用 `bun build --compile` 把这一切压成一个
可执行文件。无需安装 Node，无需在目标机器上 `npm i`，无需 Docker。只要 `./grimoire`。

## 它是怎么接起来的

一篇笔记不过是一个文件。`gray-matter` 剥离 YAML frontmatter，`@mdx-js` 编译
正文，`remark-gfm` + `rehype-slug` + `rehype-highlight` 处理表格、锚点和
语法高亮。Preact 渲染节点树；Tailwind v4 为它上色；Chart.js 绘制
图表。Bun 在构建时把这一切串联起来。

```ts
// 每篇笔记最终都会解析成这样的形状
type Note = {
  slug: string;          // "projects/grimoire"
  frontmatter: Frontmatter;
  Component: FunctionComponent; // 编译后的 MDX
  headings: Heading[];   // 提取出来用于页内目录（TOC）
};
```
