Grimoire 的笔记采用 **MDX**：标准的 GitHub 风格 Markdown，外加内联的 JSX
组件。本页是一份可供你直接复制的速查参考——下面的每个代码块都按其实际渲染的样子原样展示。

## 标题

使用 `#` 到 `######`。H1 保留给 frontmatter 中的 `title`，所以
正文内容请从 `##` 开始。

```md
## 章节
### 小节
#### 细节
```

## 内联文本

可以组合使用 **粗体**（`**bold**`）、*斜体*（`*italic*`）、***粗斜体***
（`***both***`）、~~删除线~~（`~~strikethrough~~`）以及内联 `code`
（`` `code` ``）。这类脚注式的强调可以在散文中干净地叠加使用。

## 代码块

请始终为围栏代码块标注语言，这样高亮器才会生效。下面是
三段可供对比的代码：

```ts
// TypeScript
type Note = { title: string; tags: string[]; draft?: boolean };

export function visible(notes: Note[]): Note[] {
  return notes.filter((n) => !n.draft).sort((a, b) => a.title.localeCompare(b.title));
}
```

```python
# Python
from dataclasses import dataclass

@dataclass
class Note:
    title: str
    tags: list[str]
    draft: bool = False

def visible(notes: list[Note]) -> list[Note]:
    return sorted((n for n in notes if not n.draft), key=lambda n: n.title)
```

```bash
# Bash
find notes -name '*.mdx' -print0 \
  | xargs -0 grep -L '^title:' \
  | sed 's/^/missing title: /'
```

## MDX 引入了组件

除了 Markdown 之外，你还可以把 JSX 组件直接放进散文里。Prop 用
单花括号表示表达式——`value={42}`——用双花括号表示对象
字面量——`data={{ ... }}`。

<Callout type="tip" title="组件是全局的">
笔记中无需任何 `import` 语句。像 `<Callout/>` 和
`<Badge/>` 这样的内置组件始终处于作用域内，而 `components/` 目录下的任何组件都会在构建时
自动注册。
</Callout>

内联组件同样会随文本一起流动——例如一个状态标签
<Badge color="#7c3aed">stable</Badge> 就嵌在句子内部，紧挨着一个
<Badge color="#dc2626">deprecated</Badge> 标记，而不会打断这一行。

<Callout type="warning" title="臆造的 prop 会让构建失败">
请坚持使用文档中记载的 prop 名称。未知的组件或拼错的 prop
会导致 MDX 编译失败，而不会优雅地降级处理。
</Callout>
