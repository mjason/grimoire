Transformer 是几乎所有现代语言模型和视觉-语言模型背后的架构。它的核心绝招是**注意力（attention）**：模型不再从左到右逐字阅读序列，而是让每个 token 同时审视其他所有 token，并决定哪些才重要。本文将一块块搭建起这种直觉。

## 唯一的核心思想：注意力

设想要翻译这句话："the bank raised rates."（银行上调了利率）。要判断 *bank* 指的是河岸还是金融机构，模型需要从 *rates* 中获取上下文。注意力让 *bank* 直接向句子的其余部分发起查询，并拉取那些能消除歧义的词——不需要固定窗口，也不需要循环结构。

每个 token 都被投影成三个向量：

- **查询（Query, Q）**——这个 token 正在寻找什么。
- **键（Key, K）**——这个 token 能向其他 token 提供什么。
- **值（Value, V）**——一旦它被关注，实际传递出去的内容。

一个 token 的输出，是所有 Value 的加权平均，而权重则来自它的 Query 与每个 Key 的匹配程度。

### 缩放点积注意力

```python
import numpy as np

def attention(Q, K, V):
    d_k = Q.shape[-1]
    scores = Q @ K.transpose(-2, -1) / np.sqrt(d_k)  # 相似度，经过缩放
    weights = softmax(scores, axis=-1)               # 每一行之和为 1
    return weights @ V                               # 对 value 进行加权融合
```

除以 `sqrt(d_k)` 可以防止点积随维度增大而膨胀；没有这一步，softmax 会饱和，梯度随之消失。

<Callout type="info" title="为什么要除以 √dₖ？">
如果 Q 和 K 的各个分量都是单位方差，那么它们在 `d_k` 个维度上的点积方差为 `d_k`。当 `d_k = 64` 时，标准差就是 8——大到足以让 softmax 坍缩成近乎 one-hot 的向量，从而停止学习。除以 `√d_k = 8` 可以恢复单位方差，让分布保持平滑。
</Callout>

## 多头注意力

单次注意力计算一次只能强调一种关系。**多头注意力（multi-head attention）**则并行运行多个注意力操作，每个都有自己学到的 Q/K/V 投影，最后再把结果拼接起来。

<Steps>
  <Step title="拆分">
    把输入投影成 `h` 组相互独立的 Q、K、V——对于一个 512 维、8 个头的模型，每个头都在一个 64 维的子空间中工作。
  </Step>
  <Step title="注意">
    在每个头中独立运行缩放点积注意力。某个头可能追踪句法，另一个追踪长程指代，还有一个追踪局部 n-gram。
  </Step>
  <Step title="重组">
    把 `h` 个输出重新拼接回 512 维，再经过一个最终的线性层来融合各个头。
  </Step>
</Steps>

## 位置编码

注意力是**置换不变（permutation-invariant）**的——打乱 token 的顺序，数学结果不变。这是个问题："dog bites man"（狗咬人）和 "man bites dog"（人咬狗）必须有所区别。位置编码就是把顺序信息重新注入进来。

- **正弦式（sinusoidal）**（2017 年最初的 Transformer）：把不同频率的固定正弦/余弦波加到 embedding 上。无需训练，并且能外推到未见过的长度。
- **可学习的绝对位置**：每个位置一个可训练向量（BERT、GPT-2）。简单，但受限于训练时的上下文长度。
- **旋转位置编码（RoPE）**：按位置相关的角度旋转 Q 和 K，使注意力分数依赖于*相对*距离。如今已是 LLaMA、Mistral 以及大多数当代 LLM 的标准做法。

## 编码器、解码器，还是两者兼有

最初的 Transformer 有两个堆栈；现代系统通常只选其一。

<Tabs>
  <Tab label="仅编码器">
    双向——每个 token 都能看到整个序列。非常适合理解类任务：分类、检索、生成 embedding。例子：BERT。
  </Tab>
  <Tab label="仅解码器">
    因果掩码（causal masking）——每个 token 只能看到它之前的内容，因此模型可以一次生成一个 token。这就是 GPT / LLaMA 系列以及大多数聊天模型。
  </Tab>
  <Tab label="编码器-解码器">
    编码器消化输入，解码器在生成输出的同时对编码后的源序列进行交叉注意力（cross-attention）。天然适合翻译和摘要任务。例子：T5。
  </Tab>
</Tabs>

## 一个完整的 block

每一层都用残差连接和归一化，将注意力与一个前馈网络包裹起来：

```python
def transformer_block(x):
    x = x + multi_head_attention(layer_norm(x))  # 前归一化（pre-norm）残差
    x = x + feed_forward(layer_norm(x))          # 逐位置的 MLP
    return x
```

前馈子层不过是两个线性层加一个非线性激活（通常先扩张 4 倍，例如 512 → 2048 → 512）。残差连接让梯度能够穿过数十个堆叠的 block 流动。

<Callout type="tip" title="心智模型">
剥去那些术语，Transformer 其实就是一摞"先环顾四周，再思考"的 block：注意力收集相关上下文，前馈层进行处理，残差连接让信号在深度中得以存活。其余的一切——多头、位置、掩码——都不过是围绕这个循环所做的工程设计。
</Callout>
